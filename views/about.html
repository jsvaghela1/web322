For this project, I developed an eCommerce web application that serves as the foundation for future assignments. The project involved setting up a structured development environment by creating a main application folder, initializing a Git repository, and organizing files and folders (such as views, public, and data) to house HTML, CSS, and JSON files. I implemented key features, including multiple routes that serve static content (HTML and CSS) and dynamically provide data using JSON.

I developed a "store-service" module to handle data retrieval for items and categories, ensuring functionality for accessing all items, published items, and category data. Additionally, I created server-side routes using Express.js to serve the "about" page, redirect the root route to "/about," and set up JSON-based routes for "/shop," "/items," and "/categories." To enhance user experience, I customized the static content by updating HTML links, headings, and personalized the styling through the main.css file.

Furthermore, I incorporated Bootstrap and customized CSS to enhance the design and user interface, making the app more visually appealing. I paid attention to the overall structure and organization of the app, following best practices for clean, maintainable code. After completing all functionalities, I tested the app locally, ensuring it runs without any errors before pushing it to GitHub and deploying it successfully on Vercel.

I also ensured proper error handling by implementing a custom 404 page for unmatched routes. Finally, the project was pushed to GitHub, deployed on Vercel, and all relevant submission materials were prepared and shared accordingly.
